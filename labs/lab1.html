  1. Create an object called employee that consists of three properties:

    • name
    • age
    • title
    name should be of type string, age should be of type number, and title should be of type string.

  2. Add a function to employee called printInfo() that prints the employee's information to the console, formatted like so:

    "Name: {name} - Age: {age} - Job Title: {title}"

  3. Create a subclass of employee called Manager. When an instance of Manager is created it should:

    • Automatically set the employee's title to "Manager"
    • Add a property called managementLevel to the employee, and initialize it to -1
    • Add a method called printManagementLevel() that prints a message formatted like so:

      "{name}'s is a Level: {managementLevel} Manager"

    • Add a method called setManagementLevel(), that takes a single argument and sets it as the managementLevel.
      The method must check to see if the argument is a number. If not, it should throw an exception with the following message:

      "Argument for new managementLevel is not valid"

  4. Create an instance of Manager, set its managementLevel to 10, and call its printManagementLevel method.

    Now try to call its setManagementLevel() method with a non-numeric value.

  5. Create a subclass of employee called Engineer. When an instance of Engineer is created, it should:

    • Have an Array property called projects.
    • Have a method called addProject() that takes an object with the following properties:
      • name
      • isActive
      • language

      And adds it to the projects array.

      For each property of the object that is passed in, write a validation method.

      If the property is valid, return true.
      If it is invalid, throw an exception.

      Here are the rules for each property:
      • name - Must be of type string
      • status - Must be of type boolean
      • language - Must be of type string and can only be one of the following: "JavaScript", "LISP", "C", "C++", or "Java"

      Scope the validation methods so that they cannot be called from anywhere other than the addProject() method.
      Define the validation methods so that they are at the very end of the addProject() method definition (They should automatically be hoisted by the JS compiler).

  6. The following exercise is designed to familiarize you with the JQuery selector syntax. Complete the exercise in this JSFiddle.

    Each of the <div><\div> elements wants to be assigned a color or have an event binded to it. Use JQuery to select the elements and give them their desired features.

    Refer to this JQuery selector syntax guide for guidance.

    Example: To make the text within all the <li><\li> elements red, you would write the following line of JQuery:
    $("li").css("color", "red");
